## Design Principles

1. **Information density without clutter.** Traders need data â€” show it, but with clear hierarchy.
2. **Action-oriented.** Primary actions (buy, sell, create strategy, run backtest) always prominent.
3. **Real-time feel.** Prices update in place (WebSocket), subtle animations on value changes (flash green/red).
4. **Responsive-first.** All layouts work at 480px (phone), 768px (tablet), 1024px+ (desktop).
5. **Consistent with gary-yong.com.** Same color variables, font stack, border-radius, card styling.

### Loading States

- **Page load:** Skeleton screens (grey pulsing rectangles matching layout)
- **Data refresh:** Subtle spinner in section header (not full-page overlay)
- **Backtest running:** Progress bar with "Backtesting... 45% (2020-2022)" text
- **Order submitting:** Button shows spinner, disabled until response

### Empty States

- **No portfolios:** Illustration + "Create your first portfolio to start trading" + [Create Portfolio] button
- **No strategies:** "Build your first strategy to automate trading" + [Create Strategy] button
- **No trades:** "No trades yet. Place your first order!" with arrow pointing to Trading nav link
- **No contests:** "No active contests right now. Check back soon or create your own!"

### Confirmation Dialogs

Used for destructive or significant actions:
- Delete portfolio
- Reset portfolio
- Cancel order (if > $5,000 value)
- Place order > $10,000
- Delete strategy
- Delete account
